## Commit Pattern:
* [TASK]
  * Everything that has already been set up but had to be enhanced
* [FEATURE]
  * Everything new
* [BUGFIX]
  * Fixes something that was really wrong and breaked stuff
* [WIP]
  * Well something was done, but not quite everything
